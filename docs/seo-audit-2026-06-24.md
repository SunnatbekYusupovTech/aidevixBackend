<a name="3-chuqur-seo-audit"></a>
## 3. Chuqur SEO audit — kritik tuzatishlar — commit `e788c74`

Brauzer Search-Console auditi + kod/jonli tekshiruv. "Faqat 3 sahifa indekslangan" muammosining asosiy sababi topildi: **canonical merosxo'rligi.**

### 3.1 🔴 Canonical merosxo'rligi (indekslanmaslikning asosiy sababi)

**Muammo:** root `layout.tsx` `alternates.canonical = https://aidevix.uz` o'rnatardi. O'z canonical'ini bermagan har bir sahifa buni meros olardi → `/about`, `/blog`, `/contact`, `/help`, `/pricing`, `/playground` hammasi **bosh sahifaga** canonical bo'lib, Google ularni "nusxa" deb indeksdan tashlardi.

**Tuzatish:**
- 6 ta server sahifaga o'z `alternates.canonical` qo'shildi
- Yangi layout.tsx: `mentorship`, `roadmap`, `referral`, `privacy`, `terms` (canonical + title)

**Jonli tasdiq:**
```
/about     → canonical https://aidevix.uz/about      ✅
/playground→ canonical https://aidevix.uz/playground ✅
/pricing   → canonical https://aidevix.uz/pricing    ✅
```

### 3.2 🟠 Title double-brand ("X | Aidevix | Aidevix")

**Muammo:** root template `%s | Aidevix` + sahifa o'zida `— Aidevix`/`| Aidevix` qo'shgani → ikki marta brend.

**Tuzatish:** barcha document `title:` maydonidan brend olib tashlandi (template bittasini qo'shadi). OG/twitter title'lar brend bilan qoldirildi (template ularga ta'sir qilmaydi).

Tegilgan: about, blog, contact, help, pricing, playground, careers, challenges, leaderboard, prompts, team, courses/[id], videos/[id].

**Jonli tasdiq:** `/about` → "Biz haqimizda | Aidevix" (bitta brend) ✅

### 3.3 🔴 `/videos` 404 + soft-404 video URL'lar

**Muammo:**
- `/videos` index route yo'q → HTTP 404, lekin sitemap'da bor edi
- `/videos/:id` alohida ochilganda "Video topilmadi" + `noindex` (soft-404) qaytaradi — ~57 tasi sitemap'da edi

**Tuzatish:**
- `next.config.mjs` → 301/308 redirect `/videos` → `/courses`
- sitemap'dan barcha `/videos` va `/videos/:id` olib tashlandi
- sitemap 197 → 74 toza URL

**Jonli tasdiq:** `/videos` → `308 → /courses` ✅, sitemap'da `videos=0` ✅

### 3.4 🔴 Soxta `aggregateRating` (manual action xavfi)

**Muammo:** `seedCourses.js` da reytinglar hardcoded seed (`ratingCount: 1240, 890, 2100, 3100`...) — real foydalanuvchi sharhlari emas. Course JSON-LD bu soxta sonlarni `aggregateRating` sifatida Google'ga yuborardi → "spammy structured data" / qo'lda jarima xavfi.

**Tuzatish:** `courses/[id]/layout.tsx` dan `aggregateRating` bloki olib tashlandi. Haqiqiy `CourseRating` sharhlari yig'ilgach, real `review[]` bilan qayta qo'shiladi.

**Jonli tasdiq:** kurs detal sahifasida `aggregateRating=0` ✅

### 3.5 🟠 Takror structured data (kurs detalda 2 breadcrumb)

**Muammo:** `courses/layout.tsx` (CollectionPage + ItemList + breadcrumb) `[id]` ga ham "oqib" o'tardi → kurs detal sahifasida 2 ta BreadcrumbList + keraksiz CollectionPage.

**Tuzatish:** `courses/(catalog)/` route group yaratildi — collection schema endi faqat ro'yxat sahifasiga (`/courses`) tegishli, `/courses/[id]` ga oqimaydi. `courses/page.tsx` faqat `@`-alias importlardan foydalangani uchun ko'chirish xavfsiz bo'ldi.

**Jonli tasdiq:** kurs detalda BreadcrumbList 2→1, CollectionPage 0 ✅

### 3.6 🔵 Auth/private sahifalar — noindex

**Muammo:** login, register, profile va h.k. `index:true` + bosh sahifa canonical meros olardi.

**Tuzatish:** `noindex` layout qo'shildi: `login`, `register`, `forgot-password`, `reset-password`, `verify-code`, `profile`, `subscription`, `level-up`, `bug-report`, `settings`, `/auth/*`.

**Jonli tasdiq:** `/login` → `noindex, follow` ✅

### 3.7 🟡 A11y — `maximumScale` olib tashlandi

`layout.tsx` viewport'dan `maximumScale: 5` olib tashlandi → foydalanuvchi zoom qila oladi (WCAG 1.4.4).
